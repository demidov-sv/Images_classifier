# Сервис классификации изображений 

Веб-приложение для загрузки изображений,детектирования объектов моделью **YOLOv8n** (Ultralytics) и просмотра полученного результата (списка объектов с вероятностями).
## Техническое задание по производительности

Сервис рассчитан на устойчивую работу при пропускной способности 5 картинок/секунду и поллинге результатов 25 RPS
(может быть очередь порядка 10 пользователей ожидающий результат обработки, каждый из которых пару раз в секунду делает запрос)

## Pipeline

1. Пользователь загружает изображение в форму.
2. Ассинхронный Backend сервиса проверяет изображение, сохраняет файл в S3 (MinIO) хранилище, создаёт запись в базе данных**SQlite** и ставит задачу в очередь **Redis**. Пользователь получает task_id задачи, не дожидаясь её выполнения.
3. Отдельный воркер, подключенный к Redis, берет задачи, с помощью нейросети **YOLO** находит объекты, и сохраняет результаты в базу данных, меняя статус задачи на processed
4. Клиент опрашивает `GET "/task/{task_id}"` для получения статуса обработки задачи (в случае обработанного он видит результаты)

Исходя из нагрузки был выбран следующий ## Стек технологий

API  - FastAPI, Uvicorn
Брокер - Redis 
ML -  Ultralytics YOLOv8        
База данных - SQLite
Хранилище для картинок - MinIO                          

## Быстрый старт через Docker Compose

1. Скопируйте переменные окружения

   ```bash
   copy .env.example .env
   ```

2. Положите веса модели yolov8n.pt в каталог: `YOLO_MODEL_PATH=/Backend/models/yolov8n.pt`.

3. Запуск:

   ```bash
   docker compose up --build
   ```

4. Перейдите в браузере по ссылке: **http://localhost:8000/**.

Рекомендуется использовать 2 воркера.
Для запуска 2 воркеров:
```bash
    docker compose up -d --scale worker=2
   ```

Для нагрузочного тестирования:
```bash
   docker-compose --profile test up -d locust

   ```
   Затем перейдите по ссылке и выбирайте режим тестирования
 **http://localhost:8089/**
 Выбрать number of users 30

 Для pytest
 ```bash
   docker compose exec api pytest tests/ -v

   ```


## HTTP API 

| Метод | Путь                    | Описание                                                                                      |
| ----- | ----------------------- | --------------------------------------------------------------------------------------------- |
| POST  | `/send`                 | Загрузка изображения                                                                          |
| GET   | `/`                     | Стартовая страница                                                                            |
| GET   | `/task/{task_id}`       | Получение результата статуса обработки и результата                                           |






##  Структура проекта

```text
├── Backend/                 # Бэкенд
│   ├── app/                 # Основное приложение
│   │   ├── core/            # Базовые настройки (конфиг, логирование)
│   │   ├── infrastructure/  # Подключения к внешним сервисам (БД, Redis, S3)
│   │   ├── services/        # Бизнес-логика и Pydantic-схемы
│   │   ├── main.py          # Точка входа FastAPI
│   │   ├── routes.py        # Маршруты (эндпоинты) API
│   │   └── worker.py        # Фоновый процесс обработки изображений
│   ├── models/              # Веса ML-моделей (YOLOv8)
│   ├── templates/           # HTML-шаблоны веб-интерфейса (Jinja2)
│   ├── tests/               # Unit-тесты (pytest) и нагрузочный (Locust)
│   ├── Dockerfile           # Инструкции для сборки образа API и Воркера
│   └── pyproject.toml       # Зависимости и настройки проекта (uv)
├── docker-compose.yml       # Оркестрация контейнеров 
├── pre-commit-config.yaml   # Конфигурация хуков (ruff, mypy)
└── README.md                # Документация проекта
```

## Архитектура и поток данных

                      └── Redis →  worker → YOLO → SQLite
